One of the patterns we discussed last week was the Observer pattern. What problem does this pattern solve? Explain or give an example of a situation where this pattern is useful. If you have trouble explaining in English, you can draw class diagrams and add notes instead. (10 points)

The Observer pattern is useful in a situation where events are occurring in one class, and instances of other classes want to be notified when those events occur. However, you don't know at design time what classes or instances will want to be notified. The pattern defines an Observable interface that is implemented by the class where the events happen. This interface allows other classes to register as being interested in events. The pattern also defines an Observer interface (implemented by the "interested" classes) which provides a standard way to notify the observers.
